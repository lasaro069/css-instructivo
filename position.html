fixed

        footer{
            position: fixed;
            bottom: 0;
            width: 100%;
            height: 20px;
            text-align: center;
            }

el elemento perderá sus medidas y su espacion reservado. Si lo movemos usará el elemtento posicionado contenedor como referencia. Si no lo hay, usará el elemento html de referencia. además, se quedará fijo en esa posición aunque hagamos scroll. Fija el elemento en pantalla

  • los valores top, bottom, left, right no sirven para mover el elemento, son un punto de referencia donde queremos que el elemento pase a tener un comportamiento similar al de 'fixed

            position: sticky;

z-index

Es la propiedad que nos va a permitir ordenar los elementos que se superponen

  • El número mayor del z-index es el que irá por encima
  • position debe ser relative
  • Evitar colocar números consecutivos. El código puede crecer y se necesitará modificar el z-index
  • Utilizar números distanciados para permitir libertad de selección, puede ser de 10 en 10
  • El padre no puede ir por delante del hijo,el hijo puede ir detrás del padre
  • z-index: -1 en el hijo y el padre no puede llevar z-index o debe estar en auto
    {
        position: relative;
        z-index: 0;
        }

stacking context

Contexto de apilamiento. Es el espacio donde nuestros elementos se van a ir apilando para que unos queden detrás y otros delante

De delante hacia atras:

  • elementos posicionados con z-index de valor 1 o superior
  • elementos posicionados sin z-index declarado o auto
  • Elementos no posicionados
  • Elementos con z-index negativo
